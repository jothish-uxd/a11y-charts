Accessible Charts

Chart patterns designed for perception, interaction, explanation, and implementation.

An experiment in accessible visualization by Jothish • © 2026

Charts that explain themselves.

This experiment explores how chart components can be usable beyond sight alone. The goal is not only to render charts, but to provide semantic structure, keyboard access, redundant visual encoding, and plain-language summaries that help everyone understand the data faster.

Accessibility details

These patterns are shared across the chart examples. The goal is to make accessibility part of the chart architecture, not an afterthought.

Generated summaries

Each chart provides a plain-language summary before the table so people can understand the main takeaway without inspecting every data point.

Keyboard interaction

Chart marks can be reached by keyboard. Hover behavior is mirrored for focus states so keyboard users get the same information.

Screen-reader labels

SVG titles, descriptions, and mark-level labels explain what each visual element represents.

Semantic tables

The same data is available in a real table with right-aligned numeric headers and values.

Redundant encoding

Color is paired with patterns, shapes, and stroke styles so meaning is not dependent on color alone.

DOM order

The page follows a logical reading order: title, summary, visualization, legend, and table.
